1. Basic Theme Usage

Simple theme switching in your Vue components:

<script setup lang="ts"> import { useTheme } from '@/composables/useTheme'; const { isDark, currentThemeName, toggleTheme } = useTheme(); </script> <template> <div> <p>Current theme: {{ currentThemeName }}</p> <button @click="toggleTheme">Toggle Theme</button> </div> </template>

2. Page-Specific Theme Override

Force a specific theme for certain pages (e.g., documentation always light):

<template> <!-- This page will always use light theme --> <ThemeProvider page-theme-override="light"> <v-container> <h1>Documentation Page</h1> <!-- Always displays in light theme --> </v-container> </ThemeProvider> </template>

3. Component-Level Theme Override

Force specific sections to use a different theme:

<template> <div> <!-- Normal theme section --> <v-card class="bg-general-dialog"> <v-card-title>Normal Theme</v-card-title> </v-card> <!-- Force dark theme for this section --> <ThemeProvider force-theme="dark"> <v-card class="bg-general-dialog"> <v-card-title>Always Dark Section</v-card-title> </v-card> </ThemeProvider> </div> </template>

4. Theme Controls

Provide users with theme control options:

<!-- Simple toggle button --> <ThemeToggle show-toggle-button /> <!-- Full theme mode selector --> <ThemeToggle show-mode-selector show-status /> <!-- Custom implementation --> <v-btn-toggle v-model="selectedTheme" mandatory> <v-btn value="auto">Auto</v-btn> <v-btn value="light">Light</v-btn> <v-btn value="dark">Dark</v-btn> <v-btn value="system">System</v-btn> </v-btn-toggle>

6. Conditional Styling

Apply different styles based on the current theme:

<template> <div :class="getThemeClass()"> <div class="adaptive-element"> This element adapts to the theme </div> </div> </template> <style scoped> .theme-light .adaptive-element { background: #ffffff; border: 1px solid #e0e0e0; color: #333333; } .theme-dark .adaptive-element { background: #1e1e1e; border: 1px solid #404040; color: #ffffff; } </style>

7. Navigation Integration

The theme system is integrated into the navigation drawer:

<!-- Already integrated in NavigationDrawer.vue --> <v-list-item class="border-b"> <v-list-item-title class="d-flex align-center justify-space-between"> <div class="d-flex align-center"> <v-icon icon="mdi-palette" /> <span class="ms-3">Theme: {{ currentThemeName }}</span> </div> <ThemeToggle show-toggle-button size="small" /> </v-list-item-title> </v-list-item>

8. Best Practices Summary

  • Use semantic color names: bg-general-dialog instead of bg-white
  • Provide user control: Always include theme toggle options
  • Respect system preferences: Use 'system' mode when appropriate
  • Handle special cases: Use page overrides for documentation or specific content
  • Test both themes: Ensure your content works in both light and dark modes
  • Use ThemeProvider: Wrap components that need specific theme behavior
